Como os browsers renderizam páginas web

  • A construção do DOM (Document Object Model) é o processo pelo qual o navegador transforma o código HTML bruto (bytes) em uma estrutura interativa na memória, permitindo que scripts como o JavaScript manipulem a página. Esse processo transforma tags HTML em uma "árvore de nós" (DOM Tree)

  1. Conversão (Bytes -> Caracteres) O navegador recebe os bytes brutos do arquivo HTML da rede (ou disco) e os converte em caracteres, baseando-se na codificação definida (geralmente UTF-8)
  2. Tokenização (Caracteres -> Tokens) O navegador transforma a cadeia de caracteres em tokens definidos pelo padrão HTML5 (ex: , ,

    , tags de fechamento, etc.). Os tokens funcionam como unidades semânticas de marcação.

  3. Lexing (Tokens -> Objetos/Nós) Os tokens são convertidos em "nós" (objetos no JavaScript). Cada nó representa um elemento HTML, um atributo ou um pedaço de texto
